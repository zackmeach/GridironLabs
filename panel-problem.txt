League Standings panel micro-scroll (measurement report)
======================================================

Problem statement
-----------------
The League Standings panel requires a tiny vertical scroll even though the content appears like it "should" fit.
This report records the observed runtime geometry and widget counts/heights that reproduce the issue.

Measurement method
------------------
- Launched the PySide6 UI with the packaged QSS theme applied.
- Forced the main window onto the monitor under the current mouse cursor and maximized it.
- Located the Home page's `LeagueStandingsWidget` and inspected its internal `QScrollArea` geometry:
  - viewport height
  - content widget height / sizeHint height
  - vertical scrollbar maximum/page step
- Counted and sampled heights of:
  - `StandingsRow` widgets
  - `SectionBar` widgets
- Recorded the `PanelChrome` size and its bar heights (primary/secondary/tertiary/footer) and `body_frame` height.

Runtime context
--------------
- Screen under cursor (available geometry): 2560 x 1392 at (0, 0)
- Cursor position at measurement time: (2048, 821)
- Main window geometry after maximize:
  - position: (0, -8)
  - size: 2560 x 1369

Observed behavior summarized
----------------------------
- The standings content exceeds its scroll viewport by exactly 1 pixel.
- The vertical scrollbar maximum is 1, meaning the UI allows a 1-pixel scroll range.

Impacted / relevant code paths
------------------------------
The following repo files participate in the behavior being measured:

- src/gridironlabs/ui/widgets/standings.py
  - LeagueStandingsWidget (QScrollArea + content layout)
  - StandingsRow (fixed row height)
  - StandingsHeaderRow (header row height)
- src/gridironlabs/ui/panels/panel_chrome.py
  - PanelChrome (header/body/footer stacking and body_frame sizing)
- src/gridironlabs/ui/panels/bars/standard_bars.py

- src/gridironlabs/ui/main_window.py
  - HomePage composition (constructs the League Standings panel and sets it on the grid)

  - QSS styling that affects bar heights, row/section borders, and scrollbar behavior
- src/gridironlabs/ui/app.py
  - Application bootstrapper that loads and applies the packaged QSS theme (theme selection can affect measured geometry)
- src/gridironlabs/resources/__init__.py
  - Packaged resource access helper used to read the QSS theme
